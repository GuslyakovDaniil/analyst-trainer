# Analyst LMS Pro – Тренажёр для аналитиков (Backend)

Репозиторий содержит работающий прототип (MVP) бэкенд-сервиса для веб-приложения-тренажёра, предназначенного для оценки и развития практических навыков junior- и middle-аналитиков.

Проект разработан в рамках хакатона в строгом соответствии с техническими требованиями (включая **запрет на использование Spring/Spring Boot**).

## Реализация требований кейса

**Регистрация и аутентификация:** Реализована через JWT-токены. Пароли хэшируются с помощью BCrypt.

**Тренажёры и задания:** Вывод каталога тем и тестов. Поддерживается лимит времени и количества попыток.

**Типы заданий:**
1. *Тестовые* (один правильный ответ).
2. *Поиск ошибок / SQL* (точный текстовый ответ).
3. *Открытые* (свободный текст, требует ручной проверки).

**Бизнес-логика начисления баллов:** Система оценивает ответы по единым правилам. Если в тесте есть открытые вопросы, сессия получает статус `PENDING` до проверки проверяющим.

**Прогресс пользователя:** Личный кабинет с историей сессий, лучшими результатами, бейджами (ачивками) и лидербордом.

**Дополнительный функционал (+10 баллов):** Автоматическая проверка тестовых и SQL заданий, система геймификации (достижения).
## Технологический стек

* **Язык:** Java 17
* **Web-фреймворк:** Javalin (выбран как легковесная альтернатива Spring)
* **База данных:** PostgreSQL 15
* **Доступ к данным:** Прямой JDBC, HikariCP (Connection Pool)
* **Безопасность:** JWT (JSON Web Token), jBcrypt
* **Сборка:** Maven
* **Инфраструктура:** Docker, Docker Compose
* **Frontend (для демонстрации):** Vanilla JS + HTML5 + Bootstrap 5 (встроен как статика).

## Инструкция по запуску (Docker)

Для запуска проекта вам потребуется только установленный [Docker](https://docs.docker.com/get-docker/) и Docker Compose.

1. Склонируйте репозиторий:
```bash
git clone <url_репозитория>
cd <папка_проекта>
```

2. Запустите сборку и старт контейнеров:
```bash
docker-compose up --build
```

3. После появления в логах сообщения `Система готова к работе.`, приложение будет доступно по адресу:
   👉 **http://localhost:8080**

### Тестовые доступы

При первом запуске база данных автоматически накатывает схему (`V1__init.sql`) и создает пользователя-администратора.

* **Администратор (Проверка заданий, управление тестами):**
    * Логин: `admin`
    * Пароль: `admin123`
* **Пользователь (Прохождение тестов):**
    * *Зарегистрируйте нового пользователя прямо на главной странице.*

## Архитектура и структура кода

Код строго разделён на логические слои (в соответствии с критериями качества кода):

* `config/` — конфигурация подключения к БД (HikariCP).
* `controller/` — обработчики HTTP-запросов (роутинг Javalin), валидация ввода.
* `service/` — сложная бизнес-логика (например, `TrainerService` отвечает за алгоритм автопроверки, начисление баллов и выдачу достижений).
* `dao/` — Data Access Object (чистые SQL-запросы к PostgreSQL).
* `model/` — data-классы (Java Records) для представления сущностей.
* `util/` — утилиты безопасности (генерация и валидация JWT, хэширование).

## Модель данных (База данных)

База данных спроектирована с учетом реляционных связей (3НФ):

1. **`users`** — хранит данные пользователей, их роли (USER/ADMIN) и общий рейтинг.
2. **`topics` & `tests` & `questions`** — иерархия обучающих материалов. Тесты имеют ограничения по времени и количеству попыток.
3. **`test_sessions`** — попытки прохождения. Фиксирует время начала, окончания и итоговый скор. Статусы: `IN_PROGRESS`, `PENDING`, `COMPLETED`.
4. **`test_answers`** — ответы пользователя на конкретные вопросы внутри сессии.
5. **`achievements_dict` & `user_achievements`** — словарь достижений (ачивок) и таблица связи пользователей с полученными наградами.

## Сценарии для демонстрации

1. Зайти под обычным пользователем.
2. Открыть каталог и начать тест ("Основы требований").
3. Ответить на тестовый вопрос (автопроверка) и написать текст в открытый вопрос.
4. Завершить тест (показать, что результат ушел на ручную проверку — статус `PENDING`).
5. Открыть соседнюю вкладку, зайти под `admin`.
6. Перейти в админке в раздел "Проверка", оценить открытый вопрос пользователя.
7. Вернуться к пользователю, обновить профиль: показать, что баллы зачислены, выдана "Ачивка", а пользователь появился в публичном "Рейтинге".
